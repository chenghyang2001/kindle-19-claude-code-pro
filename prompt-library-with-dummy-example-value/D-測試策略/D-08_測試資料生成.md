# D-08　測試資料生成

> **類別**：D. 測試策略

---

```
為 Article（文章） 的測試撰寫資料工廠（Factory）：
欄位：id: serial primary key, title: varchar(200) not null, content: text not null, author_id: integer references users(id), status: enum('draft','published'), created_at: timestamptz default now()
要求：
- 使用 Faker.js 生成隨機但合理的測試資料
- 提供常用的固定情境（如「已驗證使用者」「已過期訂閱」）
- 可以客製化特定欄位，其餘隨機生成
```
