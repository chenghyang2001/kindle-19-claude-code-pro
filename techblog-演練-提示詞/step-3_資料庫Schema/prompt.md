# Step 3 使用的提示詞
> 來源：`G-資料庫/G-01_Schema 設計.md`

---

```
為以下需求設計資料庫 Schema：

需求（白話文）：
使用者可以收藏文章。一個使用者可以收藏多篇文章，
一篇文章可以被多個使用者收藏。需要知道某篇文章是否被當前
用戶收藏（isBookmarked），也需要列出用戶的所有收藏（含分頁）。

資料庫：PostgreSQL（Neon Serverless），使用 Drizzle ORM

請設計：
- 資料表清單與欄位（含型別、限制、預設值）
- 外鍵關係
- 索引策略（哪些欄位需要索引）
- 考慮以下查詢模式：
  1. 查詢某用戶是否收藏了某篇文章（isBookmarked 檢查）
  2. 查詢某用戶的所有收藏（按收藏時間倒序，含分頁）
  3. 查詢某篇文章被收藏的次數（bookmarkCount）
```
