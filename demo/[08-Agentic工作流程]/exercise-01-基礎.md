# 練習 01 — 基礎：識別並行機會

## 情境說明
你有一個需要「同時做多件事」的任務，想分析哪些可以並行、哪些必須依序。

## 任務

### 任務 1：任務依賴分析
分析以下任務清單，建立依賴圖：

任務清單（某個週報自動化系統的步驟）：
- A：從 GitHub 取得本週 merged PR 清單
- B：從 Notion 取得本週新增任務
- C：對每個 PR 做重點摘要（需要 A 完成）
- D：對每個任務做狀態整理（需要 B 完成）
- E：彙整 PR 摘要 + 任務整理（需要 C 和 D 完成）
- F：生成週報 Markdown（需要 E 完成）
- G：上傳週報到 Google Drive（需要 F 完成）
- H：發送 Telegram 通知（需要 G 完成）

問題：
1. 哪些任務可以並行執行？
2. 如果用 parallel()，哪些任務適合放在同一個 parallel()？
3. 如果用 pipeline()，item 是什麼（PR？任務？）？

### 任務 2：選擇架構
根據分析，設計整體架構：
- 哪些部分用 parallel()？為什麼需要 barrier？
- 哪些部分用 pipeline()？
- 估算「全部依序」vs「最優並行」的時間差（假設每個任務需要 30 秒）

## 完成後
將依賴圖（文字描述或 ASCII 圖）+ 架構設計存入 `answer/ex01-answer.md`
