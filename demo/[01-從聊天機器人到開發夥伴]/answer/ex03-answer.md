# 練習 03 解答 — 設計第一個工作流

## 工作流目標
每天早上自動整理 `workspace/` 下所有 `.log` 檔案，移到 `logs/YYYY-MM-DD/` 資料夾。

---

## 1. 工具清單

| 工具 | 使用步驟 | 做什麼 |
|------|---------|--------|
| `Bash` | Step 1 | 執行 `date +%Y-%m-%d` 取得今天日期 |
| `Bash` | Step 2 | `find workspace/ -name "*.log"` 列出所有 .log 檔案 |
| `Bash` | Step 3 | `mkdir -p logs/YYYY-MM-DD/` 建立目標目錄 |
| `Bash` | Step 4 | `mv` 指令批次移動檔案 |
| `Bash` | Step 5 | `ls logs/YYYY-MM-DD/` 驗證移動結果 |
| `Read` | 選配 | 讀取某個 .log 確認格式，用於日誌分析 |

---

## 2. 步驟分解

```
首先，取得今天的日期（格式 YYYY-MM-DD）並建立目標目錄 logs/YYYY-MM-DD/。

然後，用 find 指令列出 workspace/ 下所有 .log 檔案的路徑清單。
   如果清單是空的，輸出「今日無 .log 檔案需要整理」並結束。

接著，用 mv 指令把每個 .log 檔案移動到 logs/YYYY-MM-DD/ 目錄下。
   （保留原始檔名，只改變目錄位置）

最後，用 ls 確認 logs/YYYY-MM-DD/ 下有正確數量的檔案，
   輸出「已整理 N 個 .log 檔案至 logs/YYYY-MM-DD/」。
```

---

## 3. 失敗點分析

### 失敗點 1：workspace/ 目錄不存在

| 項目 | 內容 |
|------|------|
| 觸發條件 | `workspace/` 目錄被刪除或路徑設定錯誤 |
| 偵測方式 | `find` 指令回傳「No such file or directory」 |
| 處理方式 | 停止執行，輸出錯誤訊息 + 正確路徑提示 |

### 失敗點 2：磁碟空間不足

| 項目 | 內容 |
|------|------|
| 觸發條件 | `logs/` 目錄所在磁碟空間不足 |
| 偵測方式 | `mv` 指令失敗，errorlevel ≠ 0 |
| 處理方式 | 立刻停止，不繼續移動，輸出已移動數量和失敗的檔案名稱 |

### 失敗點 3：檔案名稱衝突

| 項目 | 內容 |
|------|------|
| 觸發條件 | `logs/YYYY-MM-DD/` 下已存在同名 .log（同天執行兩次） |
| 偵測方式 | `mv` 覆蓋了既有檔案（無警告）或因 `-n` 旗標跳過 |
| 處理方式 | 預先檢查是否有重名，有衝突時加 `.1` `.2` 後綴 |

---

## 4. 擴展思考

**從只整理 `.log` 擴展到整理 `.log / .tmp / .bak`**

需要改動的地方：

1. **工具清單不變**，但 Bash 指令改為：
   ```bash
   find workspace/ -name "*.log" -o -name "*.tmp" -o -name "*.bak"
   ```

2. **步驟分解增加分類**：
   - 原本所有檔案進同一個目錄
   - 擴展後按類型分子目錄：`logs/YYYY-MM-DD/log/` / `.../tmp/` / `.../bak/`

3. **失敗點增加一條**：
   - 不同類型的清理規則可能不同（.tmp 可以直接刪，.bak 要保留 7 天）
   - 需要在步驟中加入「依類型決定操作」的分支邏輯

4. **設計變得更複雜**時的策略：把每種類型的整理邏輯拆成獨立的子任務，
   用 agentic 工作流的並行能力同時處理三種類型。
