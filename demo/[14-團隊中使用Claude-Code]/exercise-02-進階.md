# 練習 02 — 進階：Sub-agent 任務拆解

## 情境說明
你需要為電商後台新增「訂單報表」功能。這個功能涉及：資料庫查詢、API 設計、前端圖表、匯出 CSV、測試。

## 任務

### 任務 1：單 agent 的問題
如果讓同一個 Claude 對話完成整個訂單報表功能，會遇到什麼問題？（提示：context 長度、角色混淆、任務互相干擾）

### 任務 2：Sub-agent 任務分解
把「訂單報表功能」拆解成可以獨立執行的子任務：

**獨立性原則**：
- 子任務 A 完成後，子任務 B 才能開始（串行）
- 子任務 A 和 B 可以同時進行（並行）

畫出任務依賴圖，標明哪些可以並行：
```
[資料庫 Schema] → ?
[API 端點]     → ?
[前端元件]     → ?
[CSV 匯出]     → ?
[測試]         → ?
```

### 任務 3：子任務規格書
為「API 端點」子任務寫一份 Sub-agent Prompt，要求：
- 明確的輸入（接收什麼）
- 明確的輸出（產出什麼，格式）
- 不做什麼（邊界）
- 相依性（需要哪些前置條件）
