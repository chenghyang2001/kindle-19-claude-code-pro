# 練習 02 — 進階：核心元件實作

## 情境說明
你要用 Claude Code 實作登陸頁的兩個核心元件，並確認響應式設計正確。

## 任務

### 任務 1：Hero Section 實作
用 Claude Code 實作 Hero Section，要求：

**內容：**
- 主標題（h1）：至少 5 個字，有視覺衝擊力
- 副標題（p）：一句話說明 value proposition
- 主要 CTA 按鈕：「免費試用」
- 次要 CTA 按鈕：「查看 Demo」

**技術要求：**
- Tailwind CSS 響應式（手機單欄 / 桌面雙欄）
- 背景漸層（至少兩個顏色）
- 按鈕 hover 動畫效果

**驗收標準：**
在三個寬度下目視檢查：375px（手機）/ 768px（平板）/ 1280px（桌面）

### 任務 2：Pricing Table 實作
實作 Pricing Table 元件，包含 3 個方案：

| 方案 | 月費 | 特色 |
|------|------|------|
| Free | $0 | 基本功能，限 3 個專案 |
| Pro | $29 | 無限專案 + 優先支援 |
| Enterprise | 聯絡我們 | 自訂 + SLA 保證 |

**技術要求：**
- 中間的 Pro 方案視覺上突出（推薦標籤 + 邊框強調）
- 手機版改為垂直堆疊
- Enterprise 的 CTA 是「聯絡銷售」（連結到 mailto）

## 延伸思考
思考：如果 Pricing 的資料需要從 API 動態載入（例如根據使用者地區顯示不同幣別），你的元件設計需要什麼改變？
