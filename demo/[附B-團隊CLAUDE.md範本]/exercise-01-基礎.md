# 練習 01 — 基礎：分析現有 CLAUDE.md 範本

## 任務

### 任務 1：評分現有 CLAUDE.md
閱讀本 repo 根目錄的 `CLAUDE.md`，按以下標準評分（1-5 分）：

| 評分維度 | 定義 | 分數 |
|---------|------|------|
| 清晰度 | 規則是否清楚、不模糊 | |
| 完整性 | 是否涵蓋開發者最常問的場景 | |
| 簡潔性 | 有沒有廢話、重複、過時的內容 | |
| 可維護性 | 未來新增功能時，更新這份文件容易嗎 | |
| 有用性 | Claude 看到後，真的能給出更好的回答嗎 | |

對每個維度打分，並說明理由。

### 任務 2：找出常見 CLAUDE.md 反模式
以下是 5 個 CLAUDE.md 的常見問題，請為每個問題舉出一個壞例子和改善方式：

1. **過度詳細**：把整本 API 文件塞進去
2. **過時腐爛**：記載已經被刪除的功能或路徑
3. **角色混亂**：什麼都說「要讓 Claude 知道」但沒有優先順序
4. **禁止事項太多**：充滿「不要做 X」但不說為什麼
5. **無法驗證**：規則太抽象，Claude 看了也不知道怎麼遵守
