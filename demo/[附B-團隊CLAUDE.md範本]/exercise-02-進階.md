## 任務

### 任務 1：四種專案類型的 CLAUDE.md 差異
為以下四種專案類型，分析 CLAUDE.md 各應重點說明什麼：

|---------|---------------|
| REST API（Node.js/Python） | ? |
| 前端應用（React/Next.js） | ? |
| CLI 工具（Python/Go） | ? |
| 資料管線（Python/SQL） | ? |

### 任務 2：設計完整的 email-automation CLAUDE.md
為本 repo 中的 `email-automation-演練/` 專案設計一份 CLAUDE.md：

必須包含：
1. **專案概述**（一段話說明這是什麼）
2. **架構圖**（文字描述的目錄結構）
3. **技術堆疊**（列出使用的框架和版本）
4. **重要規則**（至少 5 條 Claude 必須遵守的規則）
5. **禁止事項**（至少 3 條 + 每條附上原因）
6. **開發指令**（如何啟動、測試、部署）

### 任務 3：CLAUDE.md 維護策略
設計一個機制確保 CLAUDE.md 不會過時：
- 什麼時候觸發 CLAUDE.md 更新？（新增目錄 / 變更技術棧 / 刪除功能）
- 如何驗證 CLAUDE.md 是否過時？（用什麼指令或方式檢查）
- 如果 CLAUDE.md 和程式碼不一致，誰優先？
