# 練習 01 解答 — 識別程式碼異味

## 任務 1：異味掃描

| 異味類型 | 具體問題 | 影響程度 |
|---------|---------|---------|
| 命名含糊 | 函式 `process`、參數 `d`/`t`/`r` 完全不說明意圖 | 高 |
| 巢狀過深 | if 巢狀達 5 層，閱讀時需要追蹤多層縮排 | 高 |
| 函式職責混雜 | 同一函式既做「過濾」又做「計算均值」又做「回傳清單」 | 高 |
| 手動迴圈取代語意化寫法 | `for i in range(len(d))` 應改 `for item in d` | 中 |
| 手動累加取代內建函式 | `total = 0; for x in result: total += x` 應改 `sum(result)` | 中 |
| 冗餘比較 | `if r == True` 應寫 `if r`；`if d[i] != None` 應寫 `if d[i] is not None` | 低 |
| 空的 else/pass | `else: pass` 完全無意義，增加閱讀雜訊 | 低 |

## 任務 2：優先排序

1. **最值得先做**：命名（`d`/`t`/`r` → `readings`/`sensor_type`/`return_average`）
   成本極低，立刻讓整個函式可讀，後續每個重構步驟都受益。

2. **最簡單快速**：移除 `else: pass` 和 `if r == True` → `if r`
   一行搞定，不改邏輯，風險為零。

3. **最危險**：拆分「過濾 vs 計算均值」兩個職責
   邊界行為（空清單回 0 / 回空 list）容易在拆解時漏掉，必須先有測試網再動。
