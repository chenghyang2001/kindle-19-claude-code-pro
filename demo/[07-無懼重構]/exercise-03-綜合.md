# 練習 03 — 綜合挑戰：大規模重構計畫

> 選配：完成前兩個練習後再挑戰

## 挑戰情境
對 `email-automation-演練/` 專案執行系統性的重構分析，產出一份可執行的重構計畫。

## 要求

1. **程式碼掃描**
   用 Claude Code 掃描 `email-automation-演練/` 的所有 .py 檔案，找出：
   - 函式長度 > 30 行的函式（列出函式名稱 + 行數）
   - 巢狀深度 > 3 層的區塊
   - 重複出現的邏輯（超過 2 個函式做類似的事）
   - 命名含糊的變數或函式

2. **優先級評估**
   對每個問題評分（1-5）：
   
   | 問題 | 影響程度 | 重構難度 | 優先級得分 |
   |------|---------|---------|----------|
   | xxx 函式過長 | 4 | 2 | 影響/難度 = 2.0 |
   | ... | ... | ... | ... |
   
   優先級得分 = 影響程度 / 重構難度（分數越高越值得優先做）

3. **重構執行計畫**
   選優先級前 3 的問題，為每個設計重構方案：
   - 採用哪個重構手法？
   - 需要先加哪些測試？
   - 預估需要多少時間？
   - 重構後預期的改善效果？

4. **風險評估**
   識別整個重構計畫的最大風險點：
   - 哪個改動最可能引入新 bug？
   - 如何降低這個風險？

## 完成標準
- [ ] 掃描結果完整（至少找到 5 個問題）
- [ ] 優先級評估有理由支撐
- [ ] 前 3 個重構計畫足夠具體可執行

## 完成後
將解答存入 `answer/ex03-answer.md`
