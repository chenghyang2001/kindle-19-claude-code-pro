# 練習 02 — 進階：程式碼品質 Checklist 實戰

## 情境說明
你完成了一個新功能，要在 PR 前做嚴格的自我審查。

## 任務

### 任務 1：建立 Checklist
根據本章的「三大品質支柱」（錯誤處理 / 安全性 / 可維護性），建立一個 10 項的程式碼品質 checklist。

格式要求（Markdown，可重複使用）：
- 每項包含：檢查項目 + 如何驗證 + 違反後果
- 按重要程度排序（最危險的放最前面）

參考項目（需自行補充完整）：
- [ ] 所有 SQL 查詢使用參數化（違反 → SQL 注入）
- [ ] 所有 API Key 從環境變數讀取（違反 → 機密洩漏）
- [ ] ...（補充到 10 項）

### 任務 2：套用到真實程式碼
選一個 `starter/` 中的程式碼（或自己寫一個 30 行的函式），用你的 checklist 逐項審查：
1. 逐項標記：通過 / 不通過 / 不適用
2. 找到的問題列清單
3. 選最嚴重的一個問題修復

## 延伸思考
思考：如果 checklist 項目太多，實際執行時容易被省略。你會如何設計讓它「不得不做」？
（提示：想想 hooks、CI 自動化）

## 完成後
將 Checklist + 審查結果存入 `answer/ex02-answer.md`
