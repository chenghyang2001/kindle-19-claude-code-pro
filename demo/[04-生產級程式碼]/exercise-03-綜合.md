# 練習 03 — 綜合挑戰：自動化品質守門

> 選配：完成前兩個練習後再挑戰

## 挑戰情境
設計一個讓品質守門「自動發生」的機制，而不是依賴開發者記憶。

## 要求

1. **設計 PreToolUse Hook**
   在 Claude Code 每次寫入 .py 檔案之前，自動執行安全性快速掃描：
   - 掃描是否有字串拼接 SQL（SQL 注入風險）
   - 掃描是否有硬編碼的 password / secret / api_key
   - 如果發現問題：停止寫入，列出問題讓開發者確認

   設計 hook 的輸出格式（讓開發者清楚知道發現了什麼）

2. **設計輸出格式**
   當發現問題時，hook 應該輸出：
   - 問題類型（SQL 注入風險 / 機密洩漏）
   - 有問題的那行程式碼
   - 建議的修正方式
   - 詢問：「是否繼續寫入？（y 繼續 / n 取消）」

3. **建立 quality-rules.md**
   建立一個 `quality-rules.md` 定義哪些 pattern 會被 block：
   - 最少 5 條規則
   - 每條規則說明：正規表達式 pattern + 為什麼危險 + 正確的替代方式

## 完成標準
- [ ] Hook 設計文件（不需要完整實作，有架構設計即可）
- [ ] 輸出格式範例（至少一個發現問題的輸出範例）
- [ ] quality-rules.md 有 5 條以上規則

## 完成後
將解答存入 `answer/ex03-answer.md`
