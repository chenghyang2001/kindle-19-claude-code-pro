# 練習 01 — 基礎：整理書中核心 Prompt 模板

## 任務

### 任務 1：整理 5 個開發場景的 Prompt 模板
從書中的範例和你自己的使用經驗，整理以下 5 個場景的 Prompt 模板：

1. **程式碼審查**：讓 Claude 按照特定標準審查程式碼
2. **除錯助手**：讓 Claude 分析 error message 並建議解法
3. **寫測試**：讓 Claude 為一個函式產生測試案例
4. **重構**：讓 Claude 在不改變功能的前提下改善程式碼品質
5. **解釋程式碼**：讓 Claude 用適合特定對象的方式解釋複雜程式碼

每個模板要包含：
- 場景說明（什麼時候用）
- Prompt 模板（含 `{{變數}}` 佔位符）
- 範例填入（一個實際例子）
- 預期輸出格式（Claude 應該用什麼格式回答）

### 任務 2：識別 Prompt 的共同模式
分析你整理的 5 個模板，找出它們的共同結構：
- 開頭通常有什麼？（角色 / 目標 / 背景）
- 中間通常有什麼？（具體指令 / 範例 / 限制）
- 結尾通常有什麼？（輸出格式 / 長度要求 / 確認語）

## 完成後
將 5 個模板 + 模式分析存入 `answer/ex01-answer.md`
