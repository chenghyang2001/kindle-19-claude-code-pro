# 練習 02 — 進階：Claude Code 生成測試

## 情境說明
你有一個複雜的業務邏輯函式，要用 Claude Code 自動生成完整的測試套件。

## 任務

### 任務 1：設計「生成測試」提示詞模板
設計一個通用提示詞模板，讓 Claude Code 為任意 Python 函式自動生成測試。

模板必須確保 Claude Code 生成：
- 至少 1 個 happy path（正常輸入）
- 至少 2 個 edge case（邊界值、空值、None）
- 至少 1 個 error case（非法輸入應拋出例外）
- 測試函式名稱要描述測試的「場景」而非「動作」

寫出完整的提示詞模板（含佔位符，讓你可以填入目標函式）。

### 任務 2：評估生成品質
用你的提示詞為 `starter/` 中的一個函式生成測試，然後評估：

評分維度（每項 1-5 分）：
1. 測試案例覆蓋是否全面（有沒有明顯遺漏的 edge case）？
2. 測試程式碼是否可以直接執行（不需要修改）？
3. 測試命名是否清楚描述場景？
4. 測試是否真的驗證了函式行為（不只是跑起來不報錯）？

記錄：哪裡好、哪裡需要人工補充。

## 延伸思考
思考：對於依賴外部服務（資料庫 / API）的函式，Claude Code 生成的測試通常會有什麼問題？你怎麼在提示詞中引導它正確處理 mock？
