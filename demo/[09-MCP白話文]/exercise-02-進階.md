# 練習 02 — 進階：MCP 整合實戰

## 情境說明
你要用 MCP 工具完成一個實際的跨服務自動化任務。

## 任務

### 任務 1：設計 MCP 呼叫序列
設計「從 Gmail 讀信 → 整理摘要 → 建立 Notion 任務」的完整 MCP 呼叫序列。

對每一步列出：
1. 呼叫哪個 MCP tool（完整名稱，例如 `mcp__gmail__search_emails`）
2. 傳入哪些參數（JSON 格式）
3. 期望收到的回應結構
4. 如果這步失敗，是否繼續還是停止？

格式：
```
Step 1: mcp__gmail__search_emails
  輸入: { "query": "is:unread subject:urgent", "max_results": 10 }
  輸出: [ { "id": "...", "subject": "...", "body": "..." }, ... ]
  失敗處理: 回傳空陣列，繼續執行

Step 2: ...
```

### 任務 2：實際執行（至少兩步）
選擇以上序列中的至少兩個步驟實際執行，記錄：
- 實際使用的參數（和設計時有什麼不同？）
- 實際收到的回應（格式和預期一致嗎？）
- 遇到的問題和解決方式

## 延伸思考
思考：如果 MCP server 的 auth token 過期，而你的 workflow 已經執行到一半，怎麼設計讓它「優雅失敗」而不是靜默產生錯誤結果？
