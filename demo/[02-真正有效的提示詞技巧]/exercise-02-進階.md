# 練習 02 — 進階：設計角色提示詞

## 情境說明
你正在開發三個不同類型的功能，需要針對每個情境設計最適合的角色提示詞。

## 任務

### 任務 1：三種角色設計
為以下三個情境各設計一個角色提示詞：

**情境 A：除錯 Python 爬蟲**
- 目標：讓 Claude 以資深 Python 工程師角度，系統性地找出 Scrapy 爬蟲的問題

**情境 B：Code Review TypeScript**
- 目標：讓 Claude 以嚴格的 TypeScript 專家角度，找出型別安全問題

**情境 C：設計 REST API**
- 目標：讓 Claude 以 API 設計師角度，考量向後相容性和版本管理

### 任務 2：測試與比較
選一個角色提示詞實際測試：
1. 先用「無角色」提示詞問同一個問題
2. 再用「有角色」提示詞問同一個問題
3. 比較兩次輸出的差異（深度、專業度、切入點）
4. 將比較結果存入 `answer/ex02-answer.md`

## 延伸思考
思考：什麼情況下「角色設定」沒有幫助，甚至可能讓輸出品質下降？
在你的實際工作中，哪個角色提示詞最值得精心設計並保存起來？
