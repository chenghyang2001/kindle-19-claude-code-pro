# 練習 01 解答 — 比較兩種工作模式

## 任務 1：兩種模式的差異

### Claude Chat（對話機器人）的行為
輸入「請幫我建立一個 hello.py」時：
- 回應：貼出 Python 程式碼文字
- **檔案沒有被建立**（只是文字輸出，需要自己手動複製貼上）
- Claude Chat 的工作邊界是「回覆訊息」

### Claude Code（開發夥伴）的行為
輸入同樣的指令時：
- 呼叫 `Write` 工具，**真的在磁碟上建立 hello.py**
- 呼叫 `Bash` 工具執行 `python hello.py` 驗證結果
- 自動完成從「建立」到「驗證」的完整流程

### 核心差異

| 維度 | Claude Chat | Claude Code |
|------|------------|-------------|
| 能力邊界 | 文字輸出 | 真實操作電腦（讀檔 / 寫檔 / 執行指令） |
| 執行結果 | 要自己複製貼上 | 直接生效 |
| 工作流程 | 對話→人工執行 | 對話→自動執行→驗證 |
| 適合場景 | 問問題、理解概念 | 開發任務、多步驟自動化 |

## 任務 2：工具呼叫識別

本次任務觀察到的工具呼叫：

| 工具 | 呼叫時機 | 做了什麼 | 少了會怎樣 |
|------|---------|---------|---------|
| `Bash` | 任務開始前 | 確認 answer/ 目錄存在 + 驗證 Python 環境 | 不影響建檔，但無法驗證環境 |
| `Bash` | 建立檔案後 | 執行 `python hello.py` 確認輸出 | 不知道程式碼是否正確執行 |
| `Write` | 中間 | 把程式碼寫入 `answer/hello.py` | 任務失敗，沒有檔案 |

### 額外觀察：Hook 攔截
嘗試直接用 `Write` 工具建立 `.py` 檔時，被 pre-tool hook 攔截（`writer-qa-iron-rule`）。
這說明 Claude Code 有自定義的行為規則，可以在特定條件下限制工具的呼叫。

---

## 心得
Claude Code 的本質不是「更聰明的對話機器人」，而是「能夠操作電腦的代理人」。
關鍵在於 **Tool Use**——有了工具，Claude 才能從「說什麼」進化到「做什麼」。
