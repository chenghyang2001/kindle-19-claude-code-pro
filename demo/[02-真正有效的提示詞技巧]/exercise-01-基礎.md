# 練習 01 — 基礎：重寫低效提示詞

## 情境說明
你的同事寫了一些低效的提示詞，導致 Claude Code 產出品質不穩定。

## 任務

### 任務 1：診斷問題
分析以下三個低效提示詞的問題所在：

- 「幫我改一下這個 function」
- 「這個 bug 怎麼修」
- 「寫個 API」

對每個提示詞，說明：
1. 缺少什麼資訊？
2. Claude Code 可能誤解的方向有哪些？
3. 會導致什麼品質問題？

### 任務 2：改寫提示詞
針對上面三個低效提示詞，各改寫一個更具體、更高效的版本。

格式要求：`[角色] + [任務] + [限制] + [輸出格式]`

範例：
- 原版：「寫個 API」
- 改版：「你是一個 Python 後端工程師。請為使用者管理系統設計 RESTful API，使用 FastAPI 框架。限制：所有端點需加 JWT 認證，回傳 JSON 格式，錯誤碼遵循 RFC 7807。」
