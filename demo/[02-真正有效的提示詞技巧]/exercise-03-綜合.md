# 練習 03 — 綜合挑戰：建立個人提示詞庫

> 選配：完成前兩個練習後再挑戰

## 挑戰情境
建立一個可以在實際工作中長期使用的「個人提示詞庫」系統。

## 要求

1. **收錄 5 個以上提示詞**
   每個提示詞包含：
   - 名稱（簡短好記）
   - 使用時機（什麼情況下觸發）
   - 完整模板（含角色 + 任務 + 限制 + 輸出格式）
   - 範例輸出（讓你知道「對的」輸出長什麼樣）

2. **設計快速調用機制**
   在 CLAUDE.md 中加入一個「快速提示詞」章節，讓 Claude Code 知道：
   當你輸入 `/debug`、`/review`、`/doc` 等縮寫時，自動套用對應的提示詞模板

3. **版本管理**
   在提示詞旁邊加入：
   - 建立日期
   - 最近修改日期
   - 修改原因（例如：「2026-06-16 加入 TypeScript 限制，因為之前輸出沒有型別」）

## 完成標準
- [ ] 至少 5 個完整提示詞，每個都有使用時機說明
- [ ] CLAUDE.md 快速調用機制設計完成
- [ ] 至少一個提示詞有版本記錄

## 完成後
將解答存入 `answer/ex03-answer.md`
