# 練習 01 解答 — 重寫低效提示詞

## 任務 1：診斷三個低效提示詞的問題

### 提示詞 A：「幫我改一下這個 function」

| 診斷維度 | 分析 |
|---------|------|
| 缺少什麼 | 哪個 function（名稱/路徑）、怎麼「改」（效能？可讀性？修 bug？）、改成什麼風格 |
| 可能誤解 | 1. 把函式整個重寫 2. 只改命名 3. 加很多不需要的功能 4. 不知道是哪個檔案的哪個函式 |
| 品質問題 | 輸出方向完全取決於 Claude 的猜測，成功率低，需要大量追問才能到位 |

---

### 提示詞 B：「這個 bug 怎麼修」

| 診斷維度 | 分析 |
|---------|------|
| 缺少什麼 | 錯誤訊息/stack trace、複現步驟、預期行為 vs 實際行為、程式碼片段 |
| 可能誤解 | 1. 不知道是哪種 bug（邏輯錯誤/型別錯誤/非同步問題）2. 可能給出不適用你環境的解法 |
| 品質問題 | Claude 只能給「通用建議」，無法給出「你的程式碼的具體修改」，幾乎沒有實用性 |

---

### 提示詞 C：「寫個 API」

| 診斷維度 | 分析 |
|---------|------|
| 缺少什麼 | 框架（FastAPI/Express/Spring）、功能（CRUD 哪些資源）、認證方式、回傳格式、錯誤處理規範 |
| 可能誤解 | 1. 選錯框架 2. 只寫一個端點還是整套 3. 沒有認證 4. 回傳格式不符合既有系統 |
| 品質問題 | 產出幾乎一定需要大幅修改，還可能引入不符合現有 codebase 風格的程式碼 |

---

## 任務 2：改寫後的高效提示詞

### 提示詞 A 改版：「幫我改一下這個 function」

**原版：** 「幫我改一下這個 function」

**改版：**
```
你是一個注重程式碼可讀性的 Python 工程師。
請重構以下 utils.py 第 12-28 行的 `parse_config` 函式：
- 目標：讓巢狀邏輯變平（最多 2 層縮排）
- 限制：不改變函式簽名（輸入/輸出型別不變）、不改變行為
- 輸出格式：只輸出修改後的函式程式碼，加上說明改了什麼地方（用繁體中文）
```

---

### 提示詞 B 改版：「這個 bug 怎麼修」

**原版：** 「這個 bug 怎麼修」

**改版：**
```
你是一個 Python 後端工程師，熟悉 requests 套件和非同步程式設計。

背景：
- 環境：Python 3.11，Windows 10
- 框架：FastAPI 0.109

問題：
呼叫 `/api/users` 端點時，有 30% 機率回傳 500 錯誤，錯誤訊息為：
  asyncio.exceptions.CancelledError: Task was cancelled

複現步驟：連續快速送出 3 個以上 POST 請求時發生

相關程式碼（附上 app/api/users.py 第 45-70 行）

請：
1. 說明最可能的根本原因（附上推理過程）
2. 提供修復方案的程式碼
3. 說明如何驗證修復是否有效
```

---

### 提示詞 C 改版：「寫個 API」

**原版：** 「寫個 API」

**改版：**
```
你是一個注重 API 設計品質的 Python 後端工程師。

請為「任務管理系統」設計並實作以下 RESTful API（使用 FastAPI 框架）：
- 資源：Task（任務）
- 端點：CRUD（建立/讀取/更新/刪除）+ 列表（含分頁）
- 認證：所有端點需要 Bearer JWT token
- 資料庫：SQLite（用 SQLAlchemy ORM）

限制：
- 錯誤格式遵循 RFC 7807（Problem Details）
- 所有回傳用 Pydantic 模型定義型別
- 不要使用 Depends() 注入以外的全域變數

輸出格式：
1. 目錄結構（app/ 下的檔案組織）
2. 完整的 main.py 和 models.py
3. 一個 API 端點的整合測試範例
```

---

## 改寫前後對比總結

| 改寫元素 | 說明 |
|---------|------|
| **角色** | 給 Claude 一個具體身份（不是「AI 助手」而是「Python 後端工程師」） |
| **背景** | 環境、版本、既有限制 |
| **任務** | 具體說明要做什麼，而不是模糊的動詞（「改」「修」「寫」） |
| **限制** | 不能做什麼（比「做什麼」更重要，限縮了錯誤空間） |
| **輸出格式** | 明確規定回答的格式，讓輸出可以直接使用 |
