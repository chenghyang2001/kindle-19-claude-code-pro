# 練習 01 — 基礎：資料庫 Schema 設計

## 情境說明
在寫任何程式碼前，先把 Freelancer Dashboard 的資料結構設計清楚。

## Freelancer Dashboard 的核心功能（需要支援）

- 管理客戶資料（姓名 / 公司 / 聯絡方式）
- 追蹤專案（專案名稱 / 客戶 / 狀態 / 預算）
- 記錄工作時間（每個專案的工時）
- 建立發票（彙整工時 → 計算金額）

## 任務

### 任務 1：資料需求分析
用白話文描述資料需求（這是 Drizzle schema 前的思考步驟）：

「接案者有多個客戶。每個客戶可以有多個專案。每個專案可以有多筆工時記錄。發票由多筆工時彙整而成，並寄給特定客戶。」

根據這個描述，識別：
- 有哪些資料表（Entity）？
- 表和表之間的關係（一對多 / 多對多）？
- 每個表需要哪些必填欄位？

### 任務 2：設計 Drizzle Schema
用 Drizzle ORM 語法設計 schema，注意：

**資料型別選擇：**
- 金額欄位用 `decimal`（不用 float，避免浮點數問題）
- 時間欄位用 `timestamp`（含時區）
- ID 用 `uuid`（不用自增 int）
- 狀態欄位用 `text` + 限制值（enum 更好但 Drizzle 實作較複雜）

**索引設計：**
- 哪些欄位會在 WHERE 條件中使用？
- 哪些欄位會做 JOIN？
- 加上適當的索引

輸出：完整的 Drizzle schema 程式碼（含 4 個資料表）
