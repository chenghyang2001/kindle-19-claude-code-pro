# 練習 03 — 綜合挑戰：完整自動化系統升級

> 選配：完成前兩個練習後再挑戰

## 挑戰情境
完成「智慧信件助理 v2」的全面升級，讓系統更智慧、更可靠。

## 要求

1. **加入重要度評分（1-5 分）**
   基於練習 02 的設計，實際修改 `email_filter.py` 加入評分邏輯。
   
   驗收：執行 `python run_briefing.py --dry-run`，確認每封信都有評分輸出。

2. **高重要度緊急通知**
   修改 `telegram_briefer.py`：
   - 4-5 分的信件：立即發送獨立的緊急通知（不等晨報彙整）
   - 1-3 分的信件：正常納入晨報
   - 緊急通知格式需與晨報格式明顯不同

3. **每週統計報告**
   新增一個 `weekly_report.py`，在每週五下午 6 點：
   - 統計本週信件數量（按評分分布）
   - 計算平均回覆時間（如果 Notion 任務有「完成日期」的話）
   - 生成 Markdown 格式的週報並發送到 Telegram

4. **Windows 排程設定**
   設定兩個 Windows schtasks：
   - 每天早上 8:00 執行 `run_briefing.py`
   - 每週五 18:00 執行 `weekly_report.py`
   
   提供完整的 schtasks 指令（可以直接複製執行）。

5. **乾跑模式驗證**
   所有新功能都必須支援 `--dry-run` 模式：
   - 不實際建立 Notion 任務
   - 不實際發送 Telegram 訊息
   - 只印出「如果正式執行，會做什麼」

## 完成標準
- [ ] 評分功能整合完成，--dry-run 可以看到評分
- [ ] 緊急通知格式與晨報格式不同
- [ ] weekly_report.py 可以執行（--dry-run 模式）
- [ ] schtasks 指令可以直接複製使用
