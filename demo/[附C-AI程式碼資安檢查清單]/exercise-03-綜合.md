# 練習 03 — 綜合挑戰：對真實 AI 生成程式碼做完整資安審查

> 選配：完成前兩個練習後再挑戰

## 挑戰情境
用你建立的資安清單，對本課程中任一個 AI 生成的程式碼做完整審查。

## 審查目標
選擇以下其中一個（建議從演練 repo 中選）：
- `email-automation-演練/` 的某個 API 端點
- `freelancer-dashboard/` 的認證流程

## 審查流程

### Step 1：確定審查範圍
列出要審查的檔案和函式（不要太多，3-5 個為主）。

### Step 2：逐項套用清單
把 exercise-02 的資安清單逐項套用到選定的程式碼：
- 打 ✅（通過）
- 打 ❌（有問題）
- 打 ⚠️（需要確認，不確定）

### Step 3：深入分析問題
對每個 ❌ 和 ⚠️：
- 說明問題的嚴重程度（Critical / High / Medium / Low）
- 說明攻擊情境（攻擊者如何利用這個問題）
- 提出修復建議（具體的程式碼改動）

### Step 4：和 Claude 對比
把你的審查結果，和「讓 Claude 審查同樣程式碼」的結果對比：
- Claude 找到了你找到的問題嗎？
- Claude 有沒有找到你沒找到的問題？
- Claude 有沒有漏掉你找到的問題？

## 完成標準
- [ ] 至少審查 3 個檔案 / 函式
- [ ] 資安清單逐項套用完成
- [ ] 和 Claude 的審查有對比分析

## 完成後
將審查報告 + 對比分析存入 `answer/ex03-answer.md`
