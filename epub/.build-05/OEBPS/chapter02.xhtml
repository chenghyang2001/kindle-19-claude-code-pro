練習 02：系統化 Debug 流程

情境：Flask API 在某些條件下回傳 500，log 只有「Internal Server Error」，沒有詳細訊息。

API 500 系統化 Debug Checklist

#步驟如何執行有發現時
1重現問題curl/Postman 用相同參數打 API，確認可穩定重現記錄最小觸發請求
2看完整 server loggrep -A 10 "500" app.log找到 traceback → 跳步驟 5
3加強 loggingPROPAGATE_EXCEPTIONS = True；開發加 DEBUG = True看到真正的 exception
4縮小觸發範圍逐一移除請求參數，二分搜尋找最小觸發條件
5讀 traceback 找根因找最深那行 File/Line去對應位置加 breakpoint
6本地 breakpoint 除錯import pdb; pdb.set_trace()觀察實際變數值
7查資料庫/外部服務直接打 DB query 或呼叫外部 API依賴項問題 → 加 timeout + fallback
8查環境差異比較 pip freeze、環境變數、config找到差異 → 統一設定

Flask Logging Decorator

import logging
import time
import traceback
from functools import wraps
from flask import request

logger = logging.getLogger(__name__)

def log_request(f):
    @wraps(f)
    def wrapper(*args, **kwargs):
        start = time.time()
        try:
            response = f(*args, **kwargs)
            elapsed = (time.time() - start) * 1000
            logger.info(
                "%s %s | params=%s | status=%s | %.1fms",
                request.method, request.path,
                dict(request.args), response.status_code, elapsed
            )
            return response
        except Exception:
            elapsed = (time.time() - start) * 1000
            logger.error(
                "%s %s | params=%s | EXCEPTION | %.1fms\n%s",
                request.method, request.path,
                dict(request.args), elapsed, traceback.format_exc()
            )
            raise
    return wrapper

高流量採樣機制

10,000 req/min 全記錄會撐爆磁碟 + 拖慢 I/O。設計原則：

場景採樣率原因
正常請求1%量太大，隨機採樣即可
500 錯誤100%錯誤不能漏掉
慢請求（>500ms）100%效能異常不能漏掉
特定使用者/IP100%（臨時）debug 特定問題時開啟
