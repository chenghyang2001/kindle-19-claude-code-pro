解讀分析

問題答案
錯誤類型ValueError — 值的格式不符合預期
錯誤位置processor.py 第 12 行（parse_date 函式內）
根本原因strptime 格式字串寫死為 %Y-%m-%d，無法處理斜線分隔的 2024/01/15
資料來源暗示2024/01/15 是 Excel 或舊系統 CSV 的典型格式，說明資料來源不只一個

Stack Trace 三要素

要素說明本例
錯誤類型例外的類別名稱ValueError
發生位置最底層 File/Line（真正爆點）processor.py:12
呼叫鏈誰呼叫誰，從上到下main.py:23 → processor.py:45 → processor.py:12

閱讀原則：從最下面那行往上看。最下面 = 真正爆點；往上 = 呼叫者。不要從第一行開始讀。

修復程式碼

import logging
from datetime import datetime
from typing import Optional

logger = logging.getLogger(__name__)

SUPPORTED_FORMATS = [
    "%Y-%m-%d",   # 2024-01-15
    "%Y/%m/%d",   # 2024/01/15
    "%d-%m-%Y",   # 15-01-2024
]

def parse_date(s: str) -> Optional[datetime]:
    for fmt in SUPPORTED_FORMATS:
        try:
            return datetime.strptime(s, fmt)
        except ValueError:
            continue
    logger.warning("無法解析日期格式：%r，回傳 None", s)
    return None

關鍵設計決策

決策原因
格式列表放函式外加格式只改一處，不動函式邏輯
try/except 在迴圈內每個格式獨立嘗試，失敗繼續下一個
Optional[datetime] 型別標注呼叫方知道可能拿到 None，強迫處理空值
