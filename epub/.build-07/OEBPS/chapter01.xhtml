練習 01：識別程式碼異味

異味掃描結果

異味類型具體問題影響程度
命名含糊函式 process、參數 d/t/r 完全不說明意圖高
巢狀過深if 巢狀達 5 層，閱讀需追蹤多層縮排高
函式職責混雜同一函式既過濾、又計算均值、又決定回傳格式高
手動迴圈for i in range(len(d)) 應改 for item in d中
手動累加total = 0; for x: total += x 應改 sum()中
冗餘比較if r == True 應寫 if r；!= None 應寫 is not None低
空的 else/passelse: pass 完全無意義，增加閱讀雜訊低

優先排序

順序問題理由
1（最值得先做）命名成本最低，立刻讓整個函式可讀，後續重構都受益
2（最簡單）移除 else:pass 和 == True一行搞定，完全不改邏輯
3（最危險）拆分職責邊界行為（空清單回 0 vs 回空 list）容易在拆解時漏掉

重構鐵律：先識別異味、排優先級，不要看到問題就立刻動手。最危險的改動必須先有測試網才能碰。
