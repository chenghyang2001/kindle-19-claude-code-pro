練習 03：自動化品質守門

PreToolUse Hook 設計

在 Claude Code 寫入 .py 前攔截，掃描即將寫入的內容：

rules = [
    (r'execute\s*\(\s*f["\'']', 'SQL 注入風險',
     '改用參數化查詢：cursor.execute(sql, (val,))'),
    (r'password\s*=\s*["\''][^"\'\']+["\'']', '機密硬編碼',
     '改用 os.environ["DB_PASS"]'),
    (r'(secret|api_key|token)\s*=\s*["\''][^"\'\']+["\'']', '機密硬編碼',
     '改用 os.environ["KEY_NAME"]'),
]
# 發現問題 → sys.exit(2) → Claude Code 停止 Write

關鍵設計決策：

  • sys.exit(2) → exit code ≠ 0，Claude Code 停止 Write
  • 只掃 .py，其他副檔名直接 sys.exit(0) 放行
  • 掃的是即將寫入的 content（PreToolUse），不是硬碟上的舊檔案

發現問題時的輸出格式

[QUALITY GATE] 發現問題，建議修正後再寫入：
  行 8 [SQL 注入風險]
    程式碼：cursor.execute(f"SELECT * FROM users WHERE id={user_id}")
    修正：改用參數化查詢：cursor.execute(sql, (val,))

  行 12 [機密硬編碼]
    程式碼：password='supersecret123'
    修正：改用 os.environ["DB_PASS"]

⛔ 寫入已中止。修正上述問題後重新產生程式碼。

quality-rules.md（5 條規則）

#Pattern危險原因正確替代
1execute\s*\(\s*f["']f-string 拼接 SQL → 注入漏洞cursor.execute(sql, (val,))
2password\s*=\s*["'][^"']+["']密碼硬編碼 → git 永久洩漏os.environ["DB_PASS"]
3(secret|api_key|token)\s*=\s*["']...機密寫死 → git 永久洩漏os.environ["SECRET_KEY"]
4except:\s*$裸 except 吞掉所有例外except pymysql.Error as e:
5subprocess.*shell=Trueshell injection 風險subprocess.run(["cmd", arg])

三層品質防線

層工具觸發時機特點
1PreToolUse hook（本練習）寫入前最早攔截，成本最低
2PostToolUse hook（Lesson 03）寫入後格式化，不阻擋
3pre-commit / CIcommit / push 後全域保障，最後防線

核心洞察：PreToolUse 比 PostToolUse 更好 — 事前阻止比事後補救便宜 10 倍。工具能做的事就不要靠人的記憶，這是生產級工程與學生作業的根本差異。
