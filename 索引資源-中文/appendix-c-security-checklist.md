# 附錄 C：AI 生成程式碼資安檢查清單

在合併任何含有身份驗證、授權、輸入處理或外部服務整合的 AI 生成程式碼前使用。每次 PR 約需 10 分鐘。

---

## 輸入驗證

- [ ] 所有輸入均已驗證類型、格式與範圍
- [ ] 具有預期格式的字串輸入在使用前已驗證
- [ ] 數字輸入設有最大長度限制
- [ ] 陣列/清單輸入設有最大長度限制
- [ ] 檔案上傳已驗證 MIME 類型、副檔名及檔案內容
- [ ] 檔案上傳已驗證檔名長度與檔案內容

## 身份驗證

- [ ] 每個受保護的端點在處理前均檢查身份驗證
- [ ] Session Token 在登出時失效——不只是登入時
- [ ] 密碼重設端點設有頻率限制
- [ ] 驗證失敗訊息統一——不區分「使用者不存在」
- [ ] 驗證失敗訊息（防止電子郵件列舉攻擊）
- [ ] 登入與重設端點均已套用頻率限制

## 授權

- [ ] 每個受保護的端點在回傳資料前均檢查擁有權/權限
- [ ] 使用者自有資源的資料庫查詢以 user_id 作為篩選條件
- [ ] 管理員端點受角色檢查保護，而非僅做身份驗證
- [ ] 存取控制邏輯中已檢查帳號停用或刪除狀態

## 資料處理

- [ ] 敏感資料（密碼、Token）不記錄於日誌
- [ ] 回傳給客戶端的錯誤訊息不含堆疊追蹤
- [ ] 機密從環境變數讀取——不硬編碼

## 外部服務

- [ ] Webhook payload 在處理前已驗證簽章
- [ ] 外部 API 呼叫設有明確的逾時值
- [ ] 外部 API 錯誤已妥善處理——不讓應用程式崩潰
- [ ] 函式庫版本已鎖定並檢查已知 CVE

## 密碼學

- [ ] 密碼使用 bcrypt/Argon2/scrypt 雜湊——不單獨使用 MD5 或 SHA-256
- [ ] 隨機 Token 使用密碼學安全的方式生成
- [ ] 所有外部通訊強制使用 TLS
